# 04_Shipping-Assistant Skill

## 适用场景
用于发货资料整理助手、FBA 箱唛、外箱贴、装箱清单、入仓单、文件匹配、单据归档、异常报告等轻量 HTML 工具。

## 项目目标
解决用户每次创建发货单后，需要整理多个发货相关文件的问题。目标是让系统辅助匹配、归档、识别关键字段、导出整理后的 Excel 或文件清单，减少人工打开文件和重复登记。

## 常见文件类型
- 装箱文件 / 装箱清单
- 外箱贴
- 物流箱唛
- FBA 箱唛
- 入仓单
- 唛头文件

## 用户当前偏好
- 不强求系统自动 OCR 识别所有内容。
- 关键字段可以手动填写，稳定优先。
- 如果自动识别会带来误判或复杂度，宁愿保留手动输入。
- 工具要轻量、离线、可直接运行。
- 不需要复杂数据库，当前以本地 HTML 工具过渡为主。

## 核心字段
发货资料整理时重点记录：
- 箱唛文件名
- 入仓单文件名
- 唛头
- 联系人
- 电话
- 地址
- PO / 发货单号
- SKU
- 箱数
- 数量
- 备注

## 文件匹配逻辑
### 基础匹配
优先按文件名中的关键词匹配：
- PO 号
- SKU
- 发货单号
- 店铺 / 站点
- 日期

### 上下文匹配
如果文件名无法完全匹配，可根据同批次上传顺序、相似关键词、文件类型进行辅助匹配。

### 异常处理
当无法确定匹配关系时，不要强行匹配。
应放入异常区，由用户手动确认。

## 装箱清单处理
如果用户表示装箱清单已由自己拆分，则系统中不需要保留多余装箱清单逻辑。
避免增加无用字段和无用模块。

## 入仓单与唛头
用户目前接受手动填写：
- 唛头
- 联系人
- 电话
- 地址

系统可提供表格回填，不强制 OCR。
如果未来需要自动识别，应作为增强功能，不影响当前稳定版本。

## 页面结构
建议结构：
1. 文件上传区
2. 文件自动分类区
3. 匹配结果表
4. 手动补充字段区
5. 异常文件区
6. 导出 Excel / 清单区

## UI 要求
- 简洁、办公风。
- 上传状态清楚。
- 文件分类清楚。
- 异常文件明显提示。
- 表格字段不要太多，优先展示实际要填的字段。
- 不需要异常报告时，允许关闭或不导出异常报告。

## 导出要求
导出的 Excel 或表格应包含：
- 文件名
- 匹配状态
- 唛头
- 联系人
- 电话
- 地址
- 备注

导出字段应和用户实际操作一致，不要加入大量无用列。

## 多人协同边界
轻量 HTML 本地工具无法天然多人共享数据库。
如果 A 上传到本地缓存，B 不会自动看到。
多人协同需要：
- 飞书协同表格
- 在线数据库
- 后端服务
- ERP 接口
当前发货助手定位为过渡期轻量工具。

## 业务痛点记录
- 产品在工厂仓、已贴标后，运营要求换标签。
- 新旧标签更替。
- 部分单据问题导致 FBA 箱唛需要重贴。
- 跟单下单前后需要快速确认三个文件是否齐全。

## 功能边界
当前优先：
- 文件分类
- 文件匹配
- 手动补字段
- 导出清单
- 异常提示

暂不优先：
- 强 OCR
- 多人实时数据库
- ERP 深度打通
- 自动生成所有物流单据

## 修复原则
- 不要为了自动化牺牲稳定性。
- 匹配不确定时宁愿提示异常，不要误匹配。
- 修改文件识别逻辑时，不能影响已有文件上传功能。
- 不允许出现“修了匹配但上传失败”的情况。

## 输出要求
生成或修改时必须给出完整 HTML。说明新增字段、匹配逻辑、异常处理、导出字段。
