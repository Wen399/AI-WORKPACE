# 02_SOP-PSI Skill

## 适用场景
用于供应链 S&OP + PSI 一体化 HTML 系统，包括库存看板、PSI 表、S&OP 表、预测质量、库存归因、采购建议、快照导出等。

## 项目目标
帮助用户把销售预测、库存、在途、工厂库存、采购跟单、补充库存整合为一个可离线运行的 HTML 经营分析系统，用于判断缺货、冗余、采购、出运、供应商交付风险。

## 核心数据源
### Sheet1：库存与到货
常见字段：
- VC 在库
- VC 理论
- VC 在途
- 海外仓库存
- 海外仓转运在途
- 出口在途
- 工厂库存
- ETA 月份

### Sheet2：销售预测主表
核心维度：
- 站点
- SKU
- 平台
- 团队
- 业务员
- 月份预测

### Sheet3：采购跟单
核心字段：
- SKU
- 工厂 / 供应商
- 已下单未生产
- 预计货好
- 确认货好
- PO
- 延期原因

### Sheet4：产品信息
核心字段：
- SKU
- 品名
- 交期
- 生产周期
- 工厂

### Sheet6：补充库存
核心字段：
- 站点
- SKU
- 发货总数
- 预计月份

### 辅助映射
- SKU 映射：硕麦 SKU ↔ 其他 SKU。
- 站点调拨范围：EU ↔ UK，US ↔ CA。
- 工厂全称 ↔ 简称映射。

## 核心口径
### 海外库存
海外库存 = 海外仓库存 + 海外仓转运在途。
不包含出口在途。

### 全链库存
全链库存 = 海外仓库存 + 出口在途 + 工厂仓库存。

### ETA 释放
出口在途必须按 ETA 月份释放，不能一次性放入当前月。
这是非常重要的历史修复点。

### 海外仓滚动
M+1 海外仓 = M 海外仓 + M 到货 - M 销售。
到货必须按 ETA 月分配。

### PSI 当月折算
如果当前月不是完整月，例如 6/8-6/30，则按剩余天数折算整月。
S&OP 不做折算。

### 库销比
库销比 = 库存 / M、M+1、M+2 三个月销售均值。
汇总时必须用总库存 / 总均值，不能简单平均 SKU 库销比。

### 采购时点
采用“赶早不赶迟”的倒推逻辑。
按交期倒推覆盖 M+2。
不能把应在 6 月下单的 SKU 放到 7 月。

### 本月采购与下月出运
用户已确认：本月采购用于下月出运。
出运量使用下个月值推导。

### 出运上限
当月出运 ≤ 当月真实可出运货源。
真实可出运货源 = 期初库存 + 当月能货好 + 当月到货 - 已分配。
不能把未来货好或未来采购提前释放。

### 调拨逻辑
同 SKU 跨站点调拨，仅限：
- EU ↔ UK
- US ↔ CA
调拨前需判断调出站点未来两个月是否仍需出货。

### SKU 映射
若销售预测 SKU 与库存 SKU 不一致，应先查映射。
不能直接丢弃或算成无库存。

## 风险归因
### 缺货风险
实际销量 > 预测销量，或未来覆盖不足。
结合供应商延期、下单不足判断。

### 冗余风险
预测下修导致库存多余。
例如原预测 800，最新预测 400，多出的库存属于冗余风险。

### 延期交付
供应商未按确认货好时间交付，且延期原因属于供应商原因。

### 风险优先级
缺货风险 > 冗余风险 > 延期交付。
正常状态不展示。

### SABC 过滤
- 缺货 / 延期可过滤 C 级。
- 冗余不能过滤 C 级，因为 C 级也可能需要清库。

## 页面要求
- 数据上传页必须清晰展示各文件状态。
- 库存看板展示库存、库销比、风险概览。
- PSI 表需要支持 SKU、站点、团队筛选。
- S&OP 表需要展示采购、出运、库存滚动。
- 预测质量页默认展示，不要求用户每次点击计算。
- 缺货快速筛选按钮必须保留。
- 业务员列优先放第一列。
- 品名必须展示，过长可截断但要能查看完整名称。

## 快照要求
- 快照必须保留 SOP 页缺货按钮。
- 快照必须保留 SKU 精准搜索。
- 快照切片器必须可用。
- 不能只修快照文件，要修正常版导出逻辑。

## 已知重点校验案例
- CY000038-1 US：库存与出运核对。
- CY000387-1 US：交期 45，7 月需出运，应 6 月下单。
- CY000239 EU：6 月发货受当月能货好限制。
- CY000364-1 US：ETA 7/8 到货、工厂库存与未生产覆盖。
- CY000180 EU：交期 45，6 月应倒推采购。
- CY000204 US：总库存与 9-10 月覆盖。
- CY000331 US：工厂库存映射错误需重点查。

## 修复原则
- 先定位问题函数，再最小修改。
- 不允许拆东墙补西墙。
- 修改一个模块后必须确认是否影响库存、采购、出运、快照。
- 不要随意改核心口径。
- 不要因为 UI 修改破坏数据逻辑。

## 输出要求
交付必须是完整 HTML 文件或明确的完整代码块。必须说明修复点、影响范围和验证方式。
