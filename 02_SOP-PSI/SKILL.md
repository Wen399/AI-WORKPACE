SOP-PSI HTML 开发工作手册（SKILL.md）
适用范围
适用于 SOP + PSI HTML 单文件系统的开发、维护、Bug 修复、功能新增、快照导出和代码检查。
AI 的任务不是重新设计系统，而是在保持既有业务逻辑的前提下，稳定修改并交付完整可运行 HTML。
一、最高优先级原则
必须遵守
业务逻辑优先于页面效果。
计算口径优先于代码优化。
旧功能稳定优先于新增功能。
修改必须最小化。
快照版必须与正式版逻辑一致。
交付必须是完整可运行 HTML。
不得擅自引入安全库存逻辑。
不得擅自改变采发货计算口径。
禁止事项
禁止：
重写整个 HTML。
为修一个 Bug 删除其它功能。
擅自改变库存、采购、出运、ETA、覆盖窗口逻辑。
用特殊判断掩盖根因。
只修 UI 不修计算逻辑。
返回伪代码、片段代码、TODO。
修改无关模块。

三、平台过滤规则
默认只分析：
Amazon-SC
Amazon-VC
禁止把非 Amazon 平台混入 PSI、SOP、库存、采购、出运计算。
四、基础维度规则
所有核心计算必须以：
站点 + SKU
作为主维度。
平台只用于过滤和辅助展示，不得作为主计算维度拆散库存逻辑。
当存在 SKU 映射时，必须先完成 SKU 映射，再进入库存、预测、采购、出运计算。
五、库存计算逻辑
1. 海外库存
海外库存必须等于：
海外库存 = 海外仓库存 + 海外仓转运在途
禁止把出口在途计入海外库存。

3

禁止把工厂库存计入海外库存。
禁止把 VC 理论库存作为海外库存。
2. 全链库存
全链库存必须等于：
全链库存 = 海外仓库存 + 海外仓转运在途 + 出口在途 + 工厂库存
如项目中已有确认口径，应以当前系统最终确认口径为准，但不得将海外库存与全链库存混用。
3. 工厂可用库存
工厂可用库存默认来源：
工厂库存 + 已下单未生产中当月可货好数量
但出运计算时，不能直接使用未来月份货好数量。
六、ETA 释放规则
出口在途必须按照 ETA 月份释放。
禁止一次性释放全部出口在途。
规则：
某月可用到货 = ETA 在该月的出口在途数量
示例：
如果 270 件 ETA 为 7 月，则只能在 7 月释放，不能在 6 月参与库存覆盖。
七、海外仓滚动库存逻辑
海外仓库存按月滚动：
4

M月期末海外仓库存
=
M月期初海外仓库存
+
M月ETA到货
-
M月预测销量
下一月：
M+1月期初海外仓库存
=
M月期末海外仓库存
如当月为当前月，预测销量需要按剩余天数折算。
八、当月预测折算规则
PSI 当月需要按剩余天数折算。
例如：
6月8日生成报表
6月剩余天数 = 6月8日 到 6月30日
当月折算销量 = 月预测销量 × 剩余天数 / 当月总天数
S&OP 不做当月折算，使用整月预测。
九、库销比逻辑
库销比默认使用未来三个月均值：
库销比 = 当前库存 / 未来三个月预测均值
汇总行不得简单平均 SKU 库销比。
汇总行必须使用：
汇总库销比 = 汇总库存 / 汇总未来三个月预测均值

十、覆盖窗口逻辑
系统采用覆盖窗口逻辑，不采用安全库存逻辑。
默认覆盖窗口：
M月出运覆盖 M+1、M+2 的需求
例如：
8月到货覆盖 8月、9月
9月到货覆盖 9月、10月
不得擅自改成安全库存模型。
十一、出运量计算逻辑
1. 出运目标
出运量的目标是补足海外未来覆盖窗口需求。
核心逻辑：
需要出运量
=
覆盖窗口内预测需求
-
覆盖窗口内预计可用海外库存
-
覆盖窗口内ETA到货
如果结果小于 0，则出运量为 0。
出运量 = MAX(0, 需要出运量)
2. 出运上限
当月出运不能超过当月真实可出运货源。
6

当月真实可出运货源
=
期初工厂库存
+
当月可货好数量
-
已分配出运数量
禁止把未来月份货好数量提前用于本月出运。
最终：
当月实际出运量
=
MIN(计划出运量, 当月真实可出运货源)
3. 已确认逻辑
“本月采购为下月出运”。
因此：
本月采购量
用于支持下月出运量
不能把本月采购直接当成本月出运。
十二、采购量计算逻辑
1. 采购量来源
采购量由未来出运需求倒推。
不是独立拍脑袋计算。
核心关系：
采购量 ≈ 未来出运量 - 当前可用工厂库存 - 已下单未生产可覆盖数量
若结果小于 0，则采购量为 0。
7

采购量 = MAX(0, 采购缺口)
2. 采购倒推月份
采购月份必须根据交期倒推。
原则：
采购月份 = 目标出运月份 - 交期周期
交期越长，下单越早。
禁止所有 SKU 使用同一个固定采购月份。
3. 交期规则
交期来自产品信息表。
如果 SKU 有明确交期，必须使用该 SKU 的交期。
如果缺失交期，不能随意猜测。
应提示缺失或使用系统默认兜底值，并在校验日志中标记。
4. 采购阈值
系统支持下单阈值：
1：一个月覆盖
0.75：三周覆盖
0.5：双周覆盖
阈值应放在数据上传页或全局配置区域。
不得新增无关 Sheet。

十四、调拨规则
调拨只允许在指定站点范围内发生。
允许范围：
EU ↔ UK
US ↔ CA
禁止跨无关站点调拨。
调拨条件：
同 SKU 或映射后同产品。
调出站点未来两个月不缺货。
调出站点有可调拨库存。
调入站点存在缺口。
调拨不得导致调出站点产生新的缺货风险。

当预测 SKU 与库存 SKU 不一致时，必须优先检查 SKU 映射。

十六、缺货判断逻辑
缺货风险判断：
未来覆盖窗口需求 > 可用海外库存 + ETA到货 + 可安排出运
或：
实际销量 > 预测销量
在预测质量模块中，实际销量大于预测销量时，应识别为缺货风险或需求超预期。
十七、冗余判断逻辑
冗余风险来自预测下修。
例如：
旧预测 = 500
新预测 = 300
预测下修 = 200
该部分可能形成冗余库存。
冗余不应简单归因为供应链问题。
10

十八、预测质量规则
APE：
APE = ABS(实际销量 - 预测销量) / 预测销量
WMAPE：
WMAPE = ΣABS(实际销量 - 预测销量) / Σ预测销量
Bias：
Bias = (实际销量 - 预测销量) / 预测销量
预测质量页默认展示，不应每次都要求用户点击计算。
十九、风险优先级
风险展示优先级：
缺货风险 > 冗余风险 > 延期交付 > 正常
正常状态默认不展示。
缺货和延期风险过滤 C 级。
冗余风险不应过滤 C 级，因为 C 级也可能需要清库。
二十、供应商延期对 SOP 的影响
延期交付会影响当月可货好数量。
如果采购跟单中确认货好时间晚于目标月份，则不得提前计入可出运货源。
延期原因中属于我方原因的：
主动延期
产品 / 新品迭代
质检排期
不应计入供应商责任交付率。
